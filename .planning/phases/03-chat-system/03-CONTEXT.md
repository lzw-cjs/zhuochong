# Phase 3 Context — 聊天系统

## 范围

Phase 3 实现用户与宠物的聊天交互：聊天引擎抽象、聊天气泡、聊天面板、交互连接。

**需求：** INT-03, INT-04, INT-05

## 设计决策

### D-01: ChatEngine 策略模式
- 定义 ChatEngine ABC（抽象基类）
- RuleBasedEngine：关键词匹配 + 模板回复
- 预留 LLM 接口（Phase 6+ 或用户自行接入）

### D-02: 关键词匹配规则
- 规则存储在 `data/chat_rules.json`
- 支持多关键词匹配（任一命中即回复）
- 未命中时返回友好兜底回复
- 规则格式：`{"keywords": ["你好", "hello"], "reply": "你好呀！"}`

### D-03: 聊天气泡增强
- 点击宠物时显示随机回复（通过 ChatEngine 生成）
- 气泡文字使用中文
- 保持 3 秒自动消失

### D-04: 聊天面板设计
- 独立窗口（非模态），可拖动
- 消息列表：用户消息右对齐，宠物回复左对齐
- 底部输入框 + 发送按钮
- 像素风配色（棕色/米色主题）
- 关闭面板不退出应用

### D-05: 消息格式
- 每条消息包含：角色（user/pet）、文本、时间戳
- 消息不持久化（会话内有效）

## Phase 2 依赖

- PetWindow.chat_requested 信号（右键菜单"聊天"）
- PetWindow.clicked 信号（点击宠物）
- ChatBubble 组件（气泡显示）
